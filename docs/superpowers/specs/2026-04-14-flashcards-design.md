# Flashcards Skill — Design Spec

**Date:** 2026-04-14
**Author:** Pierson + Claude
**Status:** Draft pending user review
**Parent spec:** `2026-04-14-marketplace-design.md`

## 1. Purpose

Generate high-quality flashcard decks from any source — a topic string, pasted notes, a textbook chapter (PDF or photo), a URL, or a lecture transcript — and export them to the formats students and teachers actually use: CSV, Anki `.apkg`, Quizlet TSV, and a printable markdown table.

**Who:** Students cramming for an exam, teachers making decks for a unit, parents helping a child study, homeschoolers building recurring review decks, and language learners (via `language-drill` chaining into this skill).

**Why:** Flashcard creation is the highest-frequency study task across grades 6+ and underserved by generic chat output. Users want a *deck file* they can drop into Anki or Quizlet, not a list in chat. This skill is the marketplace's flagship — it must be sharp, fast, and produce files that import cleanly on the first try.

## 2. Triggers

The SKILL.md description should push-trigger on phrasings like:

1. "Make me flashcards for the Krebs cycle."
2. "Turn chapter 7 of this PDF into an Anki deck."
3. "I have a spanish test friday, make me study cards from these notes"
4. "Convert this lecture transcript to Quizlet."
5. "Cloze flashcards on the Bill of Rights, please."
6. "Generate a 50-card deck on SAT vocab level 3."
7. "My kid needs help memorizing state capitals — flashcards?"
8. "export these notes as .apkg"
9. "Front/back cards for organic chemistry functional groups"
10. "can u make flashcards from this photo of my textbook"
11. "printable study cards for 3rd grade multiplication tables"
12. "Import-ready CSV of cards on French irregular verbs, A2 level."

Descriptions must trigger on both formal ("generate a flashcard deck") and informal ("flashcard this for me", "make study cards") phrasings, with and without file-format keywords.

## 3. Inputs

The skill accepts any of:

- **Topic string** — e.g. "the Krebs cycle", "AP US History Unit 3", "top 200 Spanish verbs A2".
- **Pasted notes** — free-form text of arbitrary length.
- **URL** — fetched and parsed (Wikipedia, Khan Academy, class pages).
- **PDF** — textbook chapter, handout, or lecture slides. Both text-based and scanned (OCR fallback).
- **Photo / image** — phone photo of a textbook page or handwritten notes (OCR via vision).
- **Transcript / subtitle file** — lecture audio transcripts, `.srt`, `.vtt`.

### Parameters the skill asks for (or infers)

- **Deck size** — target number of cards (default: auto-pick based on source length; offer 10/25/50/100 presets).
- **Card type** — `front-back`, `cloze`, or `mixed` (default: mixed, 70% front-back / 30% cloze).
- **Grade / level** — K-2, 3-5, 6-8, 9-12, college, grad, or CEFR A1-C2 for language decks.
- **Output formats** — one or more of `csv`, `apkg`, `tsv`, `md`. Default: all four.
- **Deck name** — inferred from source/topic if not given.
- **Tag(s)** — optional; default to `subject/topic/grade`.

## 4. Outputs

All four formats are produced by default unless the user restricts. Files land in the current working directory under `./flashcards-<slug>-<timestamp>/`.

### 4.1 `deck.csv` (universal / Anki-importable plain)

Columns (header row included):

| Column | Required | Notes |
|---|---|---|
| `front` | yes | Question / prompt. Plain text or basic HTML (`<b>`, `<i>`, `<br>`). |
| `back` | yes | Answer. Plain text or basic HTML. |
| `tags` | no | Space-separated (Anki convention). |
| `type` | no | `basic` or `cloze`. |
| `extra` | no | Hint, mnemonic, source reference. |

Cloze cards use Anki `{{c1::answer}}` syntax in the `front` column with `back` empty.

### 4.2 `deck.apkg` (Anki binary)

Generated by `shared/scripts/anki_export.py`. A valid Anki package containing:

- One deck named after the source/topic.
- Two note types: `Basic` (front/back) and `Cloze`.
- Tags as specified.
- No media in V1 (images deferred — see Open Questions).

### 4.3 `deck.tsv` (Quizlet import)

Two tab-separated columns, no header row: `term\tdefinition`. Cloze cards are flattened to front/back (the cloze sentence becomes the term, the occluded word becomes the definition) because Quizlet doesn't support cloze natively.

### 4.4 `deck.md` (printable / review in chat)

Markdown table with `| # | Front | Back | Tags |` rows. Plus a short header block: deck name, source, card count, generation date, grade level. Intended for quick review and for printing as paper flashcards (cut-and-fold).

### 4.5 Sidecar `metadata.json`

Deck name, source citation, grade level, card type distribution, generation timestamp, model version. Useful for regenerating, diffing, or re-exporting.

## 5. Workflow

Internal steps the skill runs:

1. **Parse input.** Detect input type (topic / text / URL / PDF / image / transcript). For URL, fetch and strip. For PDF, extract text (fallback to OCR if image-only). For images, run vision OCR. For transcripts, strip timestamps.
2. **Chunk source.** If >~8k tokens, chunk by section/heading. Keep a running index so citations can point back.
3. **Extract candidate facts.** Pull atomic facts, definitions, dates, formulas, vocabulary pairs. One fact = one potential card.
4. **Calibrate to grade level.** Apply the grade-level rules in `references/grade-calibration.md` to filter which facts are in-scope and how to phrase them.
5. **Generate cards.** For each fact, produce a front-back pair. Promote ~30% to cloze when the fact has a clear sentence form. Dedupe near-duplicates (cosine sim or exact-match on normalized front).
6. **Quality pass.** Drop cards with ambiguous answers, multi-answer prompts, or circular wording (front gives the answer). Enforce a max front length (~200 chars) and max back length (~400 chars).
7. **Tagging.** Assign `subject/topic/grade` tags; preserve source section headers as sub-tags when present.
8. **Write CSV + markdown.** Direct writes (deterministic).
9. **Call `anki_export.py`.** Invoke bundled script with CSV path → `.apkg`.
10. **Write Quizlet TSV.** Flatten cloze if needed, strip HTML.
11. **Write metadata.json.**
12. **Report back to user.** Summary: deck name, card count, file paths, sample 3 cards, next-step suggestion ("`open deck.apkg` to import into Anki").

## 6. Bundled scripts

Under `skills/flashcards/scripts/`:

| File | Source | Purpose |
|---|---|---|
| `anki_export.py` | symlink/copy of `shared/scripts/anki_export.py` | CSV → `.apkg` via `genanki`. Flashcards is the primary consumer; `language-drill` also chains through it. |
| `make_deck.py` | skill-specific | Orchestrator: takes parsed cards JSON, writes CSV, TSV, MD, calls anki_export. |
| `pdf_extract.py` | skill-specific (lightweight wrapper) | PDF → text + per-page index. Uses `pypdf` with OCR fallback via `ocrmypdf` or vision API. |
| `url_fetch.py` | skill-specific | URL → cleaned markdown (uses `readability-lxml` or `trafilatura`). |
| `dedupe.py` | skill-specific | Normalize + dedupe card list by fuzzy front match. |

Shared dependency: `shared/scripts/anki_export.py`. Everything else is small and lives with the skill.

The prompt logic (fact extraction, card phrasing, cloze selection, grade calibration) stays in `SKILL.md` — it's model-driven, not script-driven.

## 7. References

Under `skills/flashcards/references/` (loaded on demand, not in SKILL.md):

- `deck-naming.md` — slug conventions, tag hierarchy, handling same-topic-multiple-decks, filename timestamps.
- `cloze-syntax.md` — when to cloze vs front-back, `{{c1::...}}` vs `{{c2::...}}` multi-cloze, nesting rules, Quizlet flattening rules.
- `card-count-heuristics.md` — guidance on cards-per-chapter, cards-per-1000-words, per-topic density targets. Includes the table of recommended deck sizes by grade.
- `grade-calibration.md` — vocabulary, example type, hint phrasing, and abstraction level per grade band. Single source of truth for Section 8 of this spec.
- `card-quality-rubric.md` — what makes a good card: atomic, unambiguous, not-in-the-prompt, single-answer, minimum-info. With examples and anti-examples.
- `import-guides.md` — screenshots and step-by-step for importing into Anki, Quizlet, Remnote, and paper. Linked from the final user-facing summary.
- `language-deck-patterns.md` — patterns specific to language decks: term/definition direction, example sentences on back, audio fields (deferred), CEFR calibration.

## 8. Grade / level calibration

Card complexity, example type, and hint phrasing shift per band. Details live in `references/grade-calibration.md`; summary:

| Band | Vocabulary | Card count default | Card shape | Hints | Cloze ratio |
|---|---|---|---|---|---|
| **K-2** | ~500 most common words, no abstract terms | 10-15 | Image-friendly front, single-word back (e.g. "What color is grass?" → "green"). Print-first. | Rhyming / picture hints | 0% (cloze too abstract) |
| **3-5** | concrete nouns, basic verbs, grade-level sight words | 15-25 | Definitions, simple facts, multiplication pairs | One-word or short-phrase hint | 10% |
| **6-8** | middle-school content vocabulary, intro abstract concepts | 25-40 | Definitions, dates, formulas, vocab pairs | Mnemonic or category cue | 25% |
| **9-12** | full content vocabulary, AP-level precision | 40-60 | Multi-step facts, cause/effect, formula use, vocab in context | Source citation or related term | 35% |
| **College** | domain jargon, precise distinctions | 50-100 | Definitions with edge cases, derivations, compare/contrast | Related concept, prerequisite term | 40% |
| **Grad** | paper-citation level, narrow domain | 30-80 | Claims+evidence, method steps, paper-citable facts | Paper / author reference | 50% |
| **Language A1-C2** | CEFR-aligned wordlists | 20-60 per unit | Term↔definition bidirectional; example sentence on back | Gender/article for nouns, IPA optional | 30% (mostly sentence-cloze) |

The skill auto-detects grade from source when possible (e.g. "AP Bio chapter 12" → 11-12) and asks if ambiguous.

## 9. Evals

Five test prompts in `skills/flashcards/evals/evals.json`. Each runs the skill end-to-end and checks objective assertions.

### Eval 1 — Topic string, default everything

**Prompt:** "Make me flashcards on the Krebs cycle."

**Expected:** A deck of ~25-40 cards, grade default ≈ 9-12, all four output formats.

**Assertions:**
- `deck.csv`, `deck.apkg`, `deck.tsv`, `deck.md`, `metadata.json` all exist.
- CSV has header row with at least `front,back`.
- CSV card count is between 20 and 50.
- `.apkg` unpacks as a valid zip and contains a `collection.anki2` SQLite file.
- At least one cloze card exists (front contains `{{c1::`).

### Eval 2 — Pasted notes, explicit count and grade

**Prompt:** *(pastes ~1500 words of 7th-grade US history notes)* "Make 30 flashcards from these, 7th grade level, front-back only no cloze."

**Expected:** 30 cards, no cloze, middle-school vocabulary.

**Assertions:**
- CSV contains exactly 30 data rows.
- Zero rows have `type=cloze` or `{{c1::` in front.
- `metadata.json` reports `grade: "6-8"`.
- All fronts ≤ 200 chars; all backs ≤ 400 chars.
- No two fronts are identical after lowercasing + whitespace-collapse.

### Eval 3 — PDF input

**Prompt:** "Turn this chapter into an Anki deck." *(attaches a 12-page textbook PDF)*

**Expected:** Cards grounded in the PDF content, tags reflect chapter sections.

**Assertions:**
- `.apkg` exists and unpacks.
- `metadata.json` includes a `source` field with the PDF filename.
- At least 3 distinct tags present across cards (section-based).
- Spot-check: 3 randomly sampled card backs appear (substring match) in the PDF's extracted text.
- Card count ≥ 15.

### Eval 4 — Cloze-only, Bill of Rights

**Prompt:** "Give me cloze flashcards on the Bill of Rights."

**Expected:** All cards cloze format.

**Assertions:**
- 100% of CSV rows have `{{c1::` in the `front` column.
- Quizlet TSV still has two columns per row (cloze flattened).
- `.apkg` imports and contains cloze-type notes (inspect `collection.anki2` `notetypes`).
- Card count between 10 and 30.
- At least 10 distinct amendments/rights referenced (dedupe on subject).

### Eval 5 — Language deck, CEFR-calibrated

**Prompt:** "Flashcards on French A2 irregular verbs, 40 cards."

**Expected:** 40 term/definition pairs, bidirectional friendliness, example sentence on back.

**Assertions:**
- CSV has 40 data rows.
- Every `back` contains an example sentence (heuristic: ends with `.` and contains a space).
- `metadata.json` reports `level: "A2"` and `language: "fr"`.
- Quizlet TSV imports (open with Quizlet's importer in dry-run test script).
- Tags include `french` and `a2`.

## 10. Edge cases + failure modes

- **Huge input (>50k tokens).** Chunk, extract per chunk, merge + dedupe. Cap total output cards (default max 200). Warn the user if truncating.
- **Ambiguous topic** ("Make flashcards on functions"). Ask one clarifying question: "Math functions, programming functions, biological cell functions, or something else?" Then proceed.
- **Non-English source.** Detect language. Default: cards in source language with English glosses on back. Offer to switch.
- **PDF with images / scanned PDF.** Fall back to OCR. If OCR quality low (< ~80% confidence), surface a warning and suggest re-uploading.
- **Photo of messy handwritten notes.** OCR via vision. If <50% confidence, ask user to retype or upload a clearer photo.
- **Source with no facts (e.g. a personal essay).** Skill refuses politely: "This source is mostly narrative/opinion; flashcards work best on factual material. Want me to try anyway, or do you have notes to add?"
- **User asks for inappropriate or unsafe content** (e.g. "flashcards on how to make methamphetamine"). Refuse and offer a safe alternative.
- **Genanki / Anki library fails.** Still deliver CSV + TSV + MD. Report the `.apkg` failure with a recovery hint ("Import `deck.csv` directly into Anki via File → Import").
- **Duplicate facts across source chunks.** Dedupe pass. Keep the best-phrased variant.
- **Mnemonic/extra field overflow.** Truncate to field limit, preserve semantic core.
- **Cloze on multi-answer fact.** Use `{{c1::...}}{{c2::...}}` multi-cloze. Fall back to front-back if Quizlet is the only requested format.
- **User supplies an Anki deck already.** Out of scope in V1 — suggest `language-drill` or manual editing.

## 11. Open questions needing human decision

1. **Image/occlusion support.** Master spec defers image-occlusion. Confirm: does V1 ship *any* media support (e.g. extracting figure captions as text-only cards from PDFs), or is V1 strictly text?
2. **Anki library.** `genanki` (Python, well-maintained, MIT) vs rolling our own `.apkg` writer. Recommend `genanki` — decide before implementation.
3. **OCR backend.** Vision model (Claude native) vs `ocrmypdf` vs `pytesseract`. Vision is simpler but costlier; ocrmypdf is local. Pick one or tier by input size.
4. **Default card count per grade.** Table in Section 8 is a first pass — confirm the numbers or adjust.
5. **Cloze ratio in "mixed" mode.** Currently 30% — confirm.
6. **Quizlet flattening for cloze.** Current rule: cloze sentence → term, occluded word → definition. Alternative: skip cloze cards in TSV entirely. Decide.
7. **Deck file location.** `./flashcards-<slug>-<ts>/` in cwd vs `~/Documents/flashcards/`. Cwd is more hackable; Documents is friendlier for non-techies. Pick.
8. **Language coverage.** Which languages does V1 officially support for auto-generated decks (vs "best effort")?
9. **Regeneration flow.** If user says "more cards on this deck," do we re-run from source and merge, or generate net-new and append? Define the merge rule.
10. **Tags schema.** `subject/topic/grade` is a proposal. Validate against how actual Anki users organize decks.
11. **Graduation candidate.** Master spec flags flashcards as a possible V2 standalone CLI. Should the V1 `anki_export.py` be written with that extraction in mind (clean module boundary) or optimized purely for in-skill use?
