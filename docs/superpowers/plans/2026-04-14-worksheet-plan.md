# Worksheet Skill Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the `worksheet` skill — a Claude Code skill that generates printable PDF worksheets (student + teacher answer key) across K-12 math, reading comprehension, language drill, handwriting, and coloring-math hybrids, calibrated to grade level and rendered deterministically from HTML via Playwright.

**Architecture:** Skill is triggered by teacher/parent/tutor/student worksheet requests. Claude parses topic/grade/count/style, loads a style-specific reference, invokes deterministic Python generators under `skills/worksheet/scripts/generators/` for math and language problems (reading comp passages are generated by Claude), populates a Jinja HTML template under `skills/worksheet/scripts/templates/`, and renders to PDF via the shared Playwright-based `shared/scripts/pdf_render.py`. Worksheet is the primary driver of that shared script — API and behavior are locked here. Two-column layout is default for drills/vocab/fill-in-blank; one-column is default for word problems / reading comp / handwriting. Student and answer-key PDFs are produced as a pair.

**Tech Stack:** Python 3.11+, Playwright (Chromium) for PDF rendering, Jinja2 for HTML templating, pytest for tests, pypdf for PDF text extraction in evals. Skills run locally in the user's Claude Code session — no network required at runtime beyond the optional stock-image/branding fetch (out of V1 scope for this skill).

**Decomposition rationale:**
- `pdf_render.py` is shared but locked in this plan because worksheet is its first and most demanding consumer (custom fonts, precise margins, two-column CSS fidelity).
- Generators are split by domain (math/language/reading) so each file stays under ~400 lines and can be tested in isolation.
- Templates live next to the skill they serve, with the base template in `shared/` so `coloring-page` and `quiz-generator` can extend it later.
- Orchestration (`build_worksheet.py`, `build_answer_key.py`) is kept thin — it wires generators + template + renderer together.

# Worksheet

## When to use this skill
(list of 8-10 canonical trigger phrases copied from spec §2)

## Inputs
- Required: topic, grade/level, count (defaulted per grade if omitted)
- Optional: style, branding, layout, font size, visuals, language, accommodations, source material

## Outputs
- `<topic>-<grade>-student.pdf` (always)
- `<topic>-<grade>-answer-key.pdf` (always, unless style=handwriting)
- Optional on request: HTML (Google Docs), markdown source, combined PDF

## Workflow
1. Parse request → extract topic/grade/count/style/layout/branding/accommodations.
2. If blockers remain, ask ONE clarifying question (prefer "What grade?").
3. Load the matching reference from `references/` based on style.
4. Generate problems:
   - Math drill/word problems → `generators/math.py` (deterministic).
   - Language fill-in-blank/vocab → `generators/language.py`.
   - Reading comp → Claude writes the passage; `generators/reading.py` builds question scaffold.
   - Handwriting / coloring-math → template-driven, no generator.
5. Calibrate (problem count, difficulty, vocabulary) via `calibrate.py` — retry if miscalibrated.
6. Run `python -m worksheet.cli build --problems problems.json --style <style> --grade <grade> --out <basename>` which:
   - Fills HTML template.
   - Calls `shared/scripts/pdf_render.py` twice (student + answer key).
7. Verify files exist, page count reasonable, problem count matches.
8. Report file paths + preview snippet.

## Progressive disclosure
Load `references/<style>.md` for style-specific guidance (grade-by-grade calibration tables, common pitfalls, layout recipes). Load `references/accommodations.md` when IEP/dyslexia/ELL requested. Load `references/branding.md` when teacher/class name provided.

## Edge cases
(brief bullets — full treatment in references/)
- PreK-K: cap at 6-8 problems, force visuals.
- Bilingual: two-column side-by-side; ask if L1/L2 unclear.
- Copyrighted passages: use as-is if pasted; rewrite in style if requested "from" a book.
- >4 pages: warn before generating.
- Handwriting: no answer key.

## Constraints
- Print-safe: ≥0.5" margins, BW-safe colors.
- Never reduce content rigor for accommodations — only formatting.
- Default page size: US Letter. A4 on request.

Target length: under 300 lines.

    Returns the absolute path to the written PDF. Raises PdfRenderError on failure.
    """

```python
# shared/scripts/pdf_render.py
from __future__ import annotations
import argparse
from pathlib import Path
from typing import TypedDict, Optional
from playwright.sync_api import sync_playwright, Error as PlaywrightError

class PdfRenderError(RuntimeError): pass

DEFAULT_MARGIN = {"top": "0.5in", "right": "0.5in", "bottom": "0.5in", "left": "0.5in"}

def render_html_to_pdf(html_string: str, output_path: str, options: Optional[dict] = None) -> str:
    if not html_string or not html_string.strip():
        raise PdfRenderError("html_string is empty")
    options = options or {}
    page_size = options.get("page_size", "Letter")
    margin = options.get("margin", DEFAULT_MARGIN)
    landscape = bool(options.get("landscape", False))
    print_background = bool(options.get("print_background", True))
    wait_until = options.get("wait_until", "networkidle")

    out = Path(output_path).resolve()
    out.parent.mkdir(parents=True, exist_ok=True)

    try:
        with sync_playwright() as p:
            browser = p.chromium.launch()
            page = browser.new_page()
            page.set_content(html_string, wait_until=wait_until)
            page.pdf(
                path=str(out),
                format=page_size,
                margin=margin,
                landscape=landscape,
                print_background=print_background,
                prefer_css_page_size=False,
            )
            browser.close()
    except PlaywrightError as e:
        raise PdfRenderError(f"playwright failed: {e}") from e

    if not out.exists() or out.stat().st_size == 0:
        raise PdfRenderError("render produced no output")
    return str(out)

def _cli():
    ap = argparse.ArgumentParser()
    ap.add_argument("html_file")
    ap.add_argument("output")
    ap.add_argument("--page-size", default="Letter", choices=["Letter", "A4"])
    ap.add_argument("--margin", default="0.5in")
    ap.add_argument("--landscape", action="store_true")
    args = ap.parse_args()
    html = Path(args.html_file).read_text(encoding="utf-8")
    render_html_to_pdf(html, args.output, options={
        "page_size": args.page_size,
        "margin": {k: args.margin for k in ("top","right","bottom","left")},
        "landscape": args.landscape,
    })

if __name__ == "__main__":
    _cli()
```

    if grade not in _TABLE:

    return _TABLE[grade]

### Task 7: `build_worksheet.py` orchestrator

**Files:**

```python
def build(
    *,
    topic: str,
    grade: str,
    style: str,                # "math-drill" | "word-problems" | "reading-comp" | "handwriting" | "coloring-math" | "language-drill"
    count: int | None = None,
    problems: list[dict] | None = None,  # pre-generated; if None, dispatch to generator by style
    branding: dict | None = None,        # {teacher, class, school, logo_path, date_line, name_line}
    layout: str | None = None,           # "one-column" | "two-column"; defaults per style
    page_size: str = "Letter",
    font_size: str = "auto",
    accommodations: list[str] | None = None,
    language: str = "en",
    output_basename: str = "worksheet",
    seed: int = 0,
) -> dict:  # {"student_pdf": path, "answer_key_pdf": path|None, "problem_count": int}
```

Flow:
1. Pull `Calibration` from `calibrate.defaults_for(grade)`.
2. If `problems` is None, dispatch to generator based on `style` + `topic`.
3. Apply accommodations → mutate template vars (font size, column=1, dyslexia class, tint).
4. Render student HTML → `render_html_to_pdf`.
5. If style != "handwriting", render answer-key HTML (same template, `is_answer_key=True`) → PDF.
6. Return paths + counts.
